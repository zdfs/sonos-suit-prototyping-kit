Sonos Suit Starter Template

This starter template will help you easily prototype static HTML pages with Sonos Suit, Jade, Sass, and Grunt.

Using the template

Due to the amount of conveniences Github affords us, this project assumes you're familiar with Github and have an account.

The purpose of this project is to help you rapidly generate HTML prototypes with Sonos Suit. The project leverages Jade for HTML pre-processing (it's faster to write), Sass and Grunt for building and minifying styles and scripts. If you have questions, or need help, please reach out to suit@sonos.com.

Viewing Your Prototype

When you run the project on your local machine, you'll have access to your prototype at localhost:3000, which is great for you, but not for anyone else. In order for your pages to be viewed publicly, you'll need to put them on a server.

Github has a pretty simple way to do this with Github Pages. Let's say you forked this repo in to your own account. This fork will have two branches: master and gh-pages. If you go to pull request using gh-pages branch as your base. Complete the pull request and your pages will soon be live at your github.io url.

If you want to use your own server, you can do so. Just update whatever the url is to your own server in the package.json and run grunt deploy. You can then take the generated HTML files and upload them to the FTP server of your choice.

You can create new pages for your prototype simply by creating your own .jade files in the 'views' folder. Nested folders and views are supported and encouraged for hierarchy. If you create views while your server is running, be sure to restart the server.

Suit patterns which you can leverage are available here.

LiveReload

This project supports LiveReload, which you can activate by running grunt watch in a separate terminal window. Be sure to have the browser extension installed.
